學期專案計畫
一、 專案繳交之報告
需求規格、案立圖 (Use case diagram) 、場景圖、 Class Diagram、
基於參層架構之各案例之循序圖 (sequence diagram)、 人機介面 (User Interface)、 測試計劃。

二、 專案參考題目 

1. 學校成績管理系統
在思考此系統時，學生應已選完課，已有選課檔，此系統應有下列功能:
登錄成績、查詢成績、列印成績、計算及列印班上前五名學生名單、查核學生是否可以畢業等。

2. 學校課務管理系統
此系統應有下列功能:
準備開授之課程並可接受人工調整、幫助人工排課、學生選課併檢查衝堂 (衝堂的課不可選)等。

3. 研討會管理系統
研討會所要做的工作有:
接受投稿、安排人員審稿、接受的文章至少有一作者報名參加研討會、印出參加研討會者名單、接受報到、統計各項資料等。

4. 圖書館各種空間管理系統
圖書館的空間有大團體室、小團體室、研究小間、研討室，開放的對象有別，一次可以使用的時間也不同。選擇此題目的同學須上本校圖書館網頁，看各種空間的開放規則。

5. 圖書館館藏流通管理系統
圖書館館藏可分為可外借與不可外借兩類，讀者也分為好幾種身份，身分不同每次最多可借冊數也不同，此系統應有下列功能:
借書、還書、查詢讀者藉書狀況、借書逾期須罰錢、三天前通知所借書籍要到期等。
