假設你向 NIC 申請 ip, nic 給了一個 C 的 class 192.83.50.n 
class C 的特質為 host_id 有 254 個, 而你公司的 net_id 為 192.83.50
然因為有台北, 台中, 高雄分公司的成立, 所以你想把 254 個分成 4 組
3 組分別給三個分公司, 1 個則保留, 所以我們預計如此分:

192.83.50.  0	192.83.50. 63	台北總公司  0 和 63 (即最前和最後一個)保留, 前為網路 id, 後為 broadcast 
192.83.50. 64	192.83.50.127	台中分公司 
192.83.50.128	192.83.50.191	高雄分公司 
192.83.50.192	192.83.50.254	保留擴充用

此一觀念即為次網路定址 (subnet addressing)
外界網路到 192.83.50.n 的所有封包送到 gateway, 把這個 ip 和 255.255.255.192 (subnet mask)
AND 若為 192.83.50.0 則送台北, 192.83.50.64 送台中, 192.83.50.128 送高雄,  

255 個 ip 因為分成四組, 所以每個分公司可以分到約 64 ip
所以 1-64 的 ip 的封包是全要到台北去的, 65-128 的封包是台中去的,
如此利用二個 bit 來當成次網路 id 使用可以分出 4 組 ip

192.83.50.xx______

192.83.50.00______	台北
192.83.50.01______	台中
192.83.50.10______	高雄
192.83.50.11______	保留

而 submask 為 255.255.255.11000000 和所有的進來的封包 AND
則會得到
192.83.50.00 ______ 原要送台北的和 255.255.255.192 and 後得到 192.83.50.  0 
192.83.50.01 ______ 原要送台中的和 255.255.255.192 and 後得到 192.83.50. 64
192.83.50.10 ______ 原要送高雄的和 255.255.255.192 and 後得到 192.83.50.128


再舉一例: 
NIC 給了和春一個 B class 的 ip 126.178.nn.ee 而和春有 16 個系及行政單位要使用
所以 nn 中有 4 個 bits 要當成次網路位元, 而每個系可以分到約 2^12 個 ip
subnet mask = 255.255.11110000.0 即 255.255.240.0 (128+64+32+16)

若有二個目的封包分別要到 126.178.96.101 和 126.178.107.50 則二個和 255.255.240.0
AND 之後皆為 126.178.96.0 所以目的地相同 !
