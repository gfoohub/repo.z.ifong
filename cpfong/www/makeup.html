Hyper Text Markup Language

HTML is a language for formatting text and graphics. In addition to basic text characteristics, there are ways to generate tables, columns and graphics. No special program is required to write web pages, just a text editor or a word processor that can save in text format.

All HTML commands (also known as tags) begin with < and end with >. Most have opening and closing forms.

<b> - beginning of bold text
</b> - end of bold text

<b>ABC..xyz</b> will appear as ABC..xyz

Some tags have other options to them.

When listed in this reference, these tags will appear in this format:

<img>

align="..."

src="..."

Some tags will have many options, which will have their own groups of listed options.

A URL (Universal Resource Locator) is the address that refers to WWW page. This is how one navigates through the internet via a web browser. 

Http://server.name/path/filename.ext

Http://www.bruin.ucla.edu/alumni/programs/spirit_squad.html

server.name  - the computer the files are stored on   - www.bruin.ucla.edu
path         - the directory path to the file         - alumni/programs
filename.ext - the name of the file and its extension - spirit_squad.html
This document is meant as only a reference for the use of commands. The best way to learn how pages are written is to look at the actual web page and its scource code. To do this in netscape, first load the page, then go to the View menu and select Document source.

Some tags use an RGBhex color variable. These two charts will help you pick the right color HTMLColorChart.gif HTMLrgb2.gif

For many HTML tags, there is an indicator as to when the tag first appeared, giving some indication of how recent it is. Some HTML commands may not yet be valid in some web browsers. Changes will be made to this doument periodically to keep it current.

<frameset...>...</frameset>Frames are seperate regions of a browser that are used for displaying different pages. They are not actaully different browsers, but a set of windows that are not dependent on each other. One frame can have another go to a specific URL, while not changing its own. There is no limit to the number of frames that can be used. Frames can be setup in groups, with multiple frames with the same width having seperate heights that the next column over. Be warned that there are many users who dislike frames and may turn this option off, so be sure to provide alternatives.
Cols="N,O,*"
Sets the width of each frame in pixels. N is for the first frame, O, for the second. A seperate number is used for each frame. * sets the frame to adjust to whatever size is left available in the window.
Rows="N,O,*"
Sets height of frames in pixels. Again, * is used to indcate a variable height frame.
<noframes>...</noframes>Similar to the body tag. If the browser does not supprot frames, This tag is for enclosing an alternative for a slingle window.
<frame...>Enclosed within the tag, this will specify features for a single frame. Several of these statments within one pair of tags would be used to define a whole column or row of frames.

States the URL and/or location that will initially appear in the frame. If none is specified, a blank frame is displayed.

The width and height of the margin around the frame in N pixels.
name="framename"
The label for the frame. When doing links, the page can be loaded in a a frame specified by target="..." option in .

Scroll bar options in frames. Yes will always put scroll bars in that frame. No will never put in scroll bars. Auto will add scroll bars onlly if the page loaded into that frame is larger than the frame itself.

specifies the width of the border around a specific frame or frameset
