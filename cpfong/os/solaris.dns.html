DNS客户端的设置
1. /etc/resolv.conf文件

记录DNS服务器的地址和域名
关键字：
domainname
nameserver
# more /etc/resolv.conf
nameserver 172.16.255.3
domainname sunrise.com.cn　 //对于bind-9版本以上，可用search命令代替domain，功能更强

2. /etc/nsswitch.conf文件

记录主机名的搜索顺序等信息
修改行记录hosts:files设置为　hosts: files dns

3. nslookup命令 //dns交互测试命令，比较常用的还有dig, update

用于查询DNS服务器
用于验证客户端的设置
用于下载DNS数据库
启动时，nslookup缺省地以本地服务器为目标。响应中包括目标服务器的名字和地址。
>是提示符，这意味着可以开始发送名字服务询问或配置。
Dns客户端设置步骤：
1．vi /etc/resolv.conf
  内容如下：　nameserver xxx.xxx.xxx.xxx
              nameserver xxx.xxx.xxx.xxx
              nameserver xxx.xxx.xxx.xxx
              search xxx.xxx.com.cn
2．修改/etc/nsswitch.conf文件
　修改行记录hosts:files设置为　hosts: files dns


DNS服务器设置步骤：
DNS服务器本身做为服务端，也做为客户端，所以首先要按客户端设置步骤一样，先做那两步，然后按以下步骤做。
1，首先运行/usr/sbin/in.named ,若没有配置好，则会出现如下出借信息：can't open '/etc/named.conf'，再运行/usr/sbin/in.named Cv 看一下bind的版本号，信息如下：
in.named BIND 8.2.2-P5 Tue Jun 26 16:45:43 PDT 2001 Generic Patch-5.8-June 2001

 注：你若安装的是9版本的，缺省安装路径为/usr/local/bind,这时dns的守护进程路径为　　/usr/local/bind/sbin/named ，这时，你要创建几个文件（夹）：
　　1. mkdir /usr/local/bind/etc
   2. mkdir /usr/local/bind/var/named                //放置数据文件的位置
   3. mkdir /usr/local/bind/var/run
   4. touch  /usr/local/bind/var/run/named.pid
　  5. vi  /usr/local/bind/etc/named.conf　　　　　　//以后的步骤与bind8的一样
 
2，创建/etc/named.conf文件，在里面输入如下信息：
　　options {
   directory "/var/named";   //数据库文件的位置，要创建named这个目录，
            };
zone "." IN {
type hint;
file "named.ca";
};
zone "0.0.127.in-addr.arpa" IN {
type master;　　　         //这里master表示为主dns server,若作为辅助，则变为slave
file "named.local";　　　　//还要在此行下面添加如下一行：masters { 192.9.207.11; };
allow-update { none; };　　
};
zone  "abc.com" IN {
   type master;  　　　　　　//这里master表示为主dns server
   file "named.abc.com";    //这里的abc.com就你你要创建的域，正向解释
   allow-update { none; };
   };
zone  "13.12.11.in-addr.arpa" IN {
   type master;
   file "named.13.12.11";　　　//反向解释，如你的ip为11.12.13.1，这里为创建反向解释
   allow-update  { none; };　　//域，13.12.11，
   };
这里要注意，必须至少要有这四个zone，若你要添加其它的，只要再创建新的zone就行了，比如你要新添efg.com这个域，ip为　23.34.56.1，你就要新添zone “efg.com”及反转域56.34.23了，这里不详细说明的，若为B级地址，反向解释文件名要变为“named.12.11”了，反向解释的PTR记录可这样写：1.13 IN PTR www.abc.com.
3，在/var/named目录下创建如下几个数据库与/etc/named.conf文件中相对应的文本数据库
1. named.ca ;2. named.local ;3. named.abc.com ;4. named.13.12.11

named.local文件内容如下：
$TTL86400
@       IN      SOA     www.abc.com. root.www.abc.com.  (
                                    2002070707 ; Serial  //注:每改动一下这四个文件，都要修改这个字段一次
                                    28800      ; Refresh
                                    14400      ; Retry
                                    3600000    ; Expire
                                    86400 )    ; Minimum
            IN      NS     www.abc.com. 　　　　　//代表授权的dns　server
1       IN      PTR     localhost.
named.abc.com文件内容如下：
$TTL86400
@       IN      SOA     www.abc.com. root.www.abc.com.  (

到这里，主体的工作完成了，现在进行测试，运行in.named进程（bind9为named进程）#/usr/sbin/in.named
若出错，则说明以上的配置文件有错，请仔细检查，并改正。
如果已经运行了in.named进程，可用下列命令进行重读数据文件：　　
#ps Cef |grep in.named　　记下pid
#kill CHUP pid
#nslookup
>
Default Server:  www.abc.com
Address:  11.12.13.1
>
到此，则说明你已经配置好了你的dns　server，对于虚拟主机的应用，还要在named.abc.com文件中添加CNAME记录，还要修改apache的配置文件等，具体做法这里不说了，会在后面详细说明配置虚拟主机的应用。
