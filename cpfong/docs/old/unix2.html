UNIX 的基本指令



壹、前言

        UNIX和網路的關係非常密切，雖然使用網路不一定要用UNIX，不過他卻
    是一個相當好的選擇。UNIX作業系統可以讓多個使用者同時連線使用，也可
    以透過網路共享檔案資源，也可以和其他在網路上的使用者互通有無，或者
    和同在網路上的友人互通訊息聯絡情感。
        SUN OS是SUN 所使用的UNIX作業系統，目前本中心所使用的便是SUN OS 
    4.1.3_U1的版本，透過學習SUN OS的使用指令，使用者將可更得心應手的使
    用電腦資源。


貳、環境說明

    本文範例之操作基台為 sparc( 140.130.1.8 )，自586電腦教室，電腦連線
    操作。


參、操作內容



一、進出系統及密碼設定



1. 進入UNIX系統


    要進入系統時，首先輸入正確的使用者名稱(USER ID)及密碼(PASSWORD) ，
    即可進入sparc:/home1/axxxxxx>的提示符號。
        如第一次進入你的UNIX帳號，則不用輸入密碼，等進入系統後，自行設
    定。

    以下以進入sparc (140.130.1.8) 工作站為例：

     BBS Users login : bbs
     -----------------------------------------------------------------



     SunOS UNIX (sparc)

     login: a207105            <----- 請輸入你的帳號
     Password:                 <----- 請輸入你的密碼（第一次使用，不需
                                      輸入）
     Last login: Wed Nov 13 12:06:21 from 140.130.3.5
     SunOS Release 4.1.3_U1 (GENERIC) #1: Tue May 14 13:49:34 CST 1996
                               <----- 以上兩行為系統資訊
     sparc:/home1/a207105>     <----- Sun 的系統符號
     若輸入錯誤則會出現下列訊息：
     Login incorrect
     login:

    請再重覆輸入帳號及密碼。若輸入正確但卻不能進入，則請與機房管理者諮
    詢。

 　 說明：UNIX將大寫與小寫視為不同的字，所以輸入時請小心。



2. passwd 設定密碼與更改密碼

　      密碼須以英文字母為開頭，長度必須介於6至8個字。為安全起見，請至
    少一個月更改一次密碼。


    (1) 設定密碼


    第一次使用請務必設定密碼，設定密碼的方法如下。
 　　sparc:/home1/a207105>passwd     <----- 輸入指令
     Changing password for a207105 on sparc.
     New password:                   <----- 輸入密碼，為了保密，所以不會顯視
     Retype new password:            <----- 為了確認，所以再輸入一次密碼
     如果兩次所輸入的密碼不同，則會出現....
     Mismatch - password unchanged.
     sparc:/home1/a207105>
     請再請再重新設定密碼方可。

    (2) 更改密碼


     更改密碼時，請先輸入舊密碼、再輸入二次新的密碼。
     sparc:/home1/a207105>passwd     <----- 輸入指令
     Changing password for a207105 on sparc.
     Old password:                   <----- 輸入舊密碼
     New password:                   <----- 輸入新密碼
     Retype new password:            <----- 為了確認，所以再輸入一次新密碼
     sparc:/home1/a207105>           <----- 更改密碼完成


3. logout 離開系統

     sparc:/home1/a207105>logout

3. chmod 更改使用權限

        前面說過，UNIX是一個多人多工式得作業系統，所以在檔案處理方面和
    DOS 有很大的不同點，在UNIX上，每一個檔案或目錄都有一個使用者，和一
    個GROUP ，還有OWNER 和GROUP 以外的人，這些人對該檔案的讀寫權限，都
　　經由該檔案的模式所控制，以下就來介紹更改檔案模式的指令 -- chmod 。

    格式：chmod 權限 檔案名稱或目錄名稱

    使用權限分為三類：讀(r)、寫(w)、執行(x)。
    rwx rwx rwx
    ^^^ ^^^ ^^^
    │   │  └ other
    │   └ group
    └ user
    如上圖所示，以三個字元為一組，分別代表自己 (user)、所屬群組(group)
    及其他人(other)的權限。


    使用者類別以u、g、及a 表示
         u：user 表檔案的擁有者
         g：group 同一個 group 者
         o：others 不同 group 的其他人
         a：all  上述三項 u,g,o 之總合

    更改使用權限時可以以=、-、+ 來更改
         =：設定權限
         -：除去某項權限
         +：加上某項權限

    舉例如下：
     sparc:/home1/a207105>chmod a=rwx test    ----> 將所有人的使用權限
                                                    設為 rwx
     sparc:/home1/a207105>ls -l tess
     -rwxrwxrwx  1 a207105         5 Nov 13 12:42 test

     sparc:/home1/a207105>chmod g-rwx,o-rwx test 
     sparc:/home1/a207105>ls -l test
     -rwx------  1 a207105         5 Nov 13 12:42 test

     sparc:/home1/a207105>chmod g+x,o+r test
     sparc:/home1/a207105>ls -l test
     -rwx--xr--  1 a207105         5 Nov 13 12:42 test

    另外一種表視方法為使用權限代號去代表owner、group、other的限
        例：
    檔案屬性:      －　ｒｗ－　－ｗ－　ｒ－ｘ
    二進位表示法:      1 1 0   0 1 0   1 0 1    
    八進位表示法:        ６　　　２　　　５
                       owner   group   other
    
      sparc:/home1/a207105>chmod 750 test
      sparc:/home1/a207105>ls -l test
      -rwxr-x---  1 a207105       145 Nov 13 13:18 test
       111101000
        7  5  0 ----> 所以檔案模式是使用750
       
      sparc:/home1/a207105>    


4. cat, more, head, tail 列出檔案內容


     cat filename      <----可列出全部檔案內容
     more filename     <----可列出全部檔案內容，但會自動分頁，可按
                            "space bar"繼續。
     head -n filename  <----可列出前 n 行。
     tail -n filename  <----可列出後 n 行。

    舉例如下：
sparc:/home1/a207105>cat test
English: The golden willow by the river is the bride in the seeting sun.
Chinese: 那河畔的金柳是夕陽中的新娘
                                                 joyen (無敵夢幻小笨球)
sparc:/home1/a207105>head -1 test
English: The golden willow by the river is the bride in the seeting sun.


sparc:/home1/a207105>tail -1 test
                                                 joyen (無敵夢幻小笨球)



5. 建檔（cat > filename）


        以cat 建檔時，不能修改上一行的資料。結束編輯時，請在新的一行行
    首打入^D（ 表 "ctrl" + "d" 兩鍵）
　  另 SunOs 提供一編輯器 vi，操作說明請見　vi 操作說明。
     sparc:/home1/a207105>cat > joyen
     This is a test for file creation with cat.
     You can try yourself.
     ^D   
     sparc%

     sparc:/home1/a207105> cd ..         <---- 代表回到上一層目錄

     sparc:/home1/a207105> rmdir mywork

     而你的螢幕則會進入 talk 的視窗，在左上角會出現下列訊息：

     [Ringing your party again]
     [Ringing your party again]

     sparc:/home1/a207105>talk a207105@sparc.nypi.edu.tw

     若連線成功則會出現下列訊息：

     [Connection established]

    說明：如果使用 talk 指令的兩人均在同一主機上 (例如均在sparc上)，則
          hostname 可省略不打。
