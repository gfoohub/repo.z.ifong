何謂 Samba
  • 簡單來說，Samba 就是一套程式能讓你的 Unix 系統認識 SMB （Server Message Block）協定
  • 也就是說，裝上 Smaba 之後，你將可以提供檔案共享跟印表機的服務

1.1 Samba是什麼?

Samba是由許多程式所組成的一套軟体，而這一套軟体中的程式可共同執行工作，進而允許Client透過SMB(Server Message Block)協定，早期Samba是專為Unix系統所寫，但是現在的Samba可以執行在Netware，OS/2 and UMS.

實際上，這意謂著你可以從Lan Manager 客戶端重導磁碟和印表機到Unixi or Windows for workgroups 3.1 or Windows Clients, Linux Clients and O/S 2 Clients 的磁碟和印表機，也有一部份Unix一般的程式也成為Samba的一部分，允許Unix使用者使用類似FTP介面可操作檔案空間和印表機，只要是伺服器上SMB服務即可Samba使得以上所提的這些系統有能力，扮演像一個Lan Manager or Windows NT Server的角色，而且只是利用新增功能並且可達到專為系統管理者操作與管理便利而設計。 ?

以摘要的方式一一列舉這套軟体的組成份子:

  • smbd, 是一部SMB 伺服器. 是實際處理來自於客戶端的連接，處理檔案，授權和使用者名稱的工作
  • nmbd, 這是Netbios 名稱伺服器, 專門協調客戶端找出伺服器的位置，進行瀏灠工作和管理網域，這些功能已內建在Samba裡面。
  • smbclient, 這是Unix主機的客戶端程式
  • smbrun, 是一個小小的附加程式，協助伺服器執行外部程式。
  • testprns, 這是一個測試伺服器存取印表機的程式。
  • testparms, 這是一個測試Samba組態檔案的正確性
  • smb.conf, 此為Samba的組態檔
  • smbprint, 此為一個範例批次執行檔，允許Unix主機使用smbclient將列印工作送給SMB伺服器。。
  • 文件! 千萬別忽視去看它喔，你可以省下許多時間

1.2 Samba現在的版本是多少呢?

目前最新的版本是2.0.6，假如你要查看版本改變的記錄檔請至此： ftp://www.samba.org

1.3 到那裡取得？

Samba suite可以從samba.org以匿名方式取得最新的版本：目錄位置/pub/samba/alpha 要注意的是以上不括任何二進位的形式，Samba只可以source form(來源形式散發)，雖然二進位從其他網站是可以的，最近一些FreeBSD散發出去的版本，例如，Samba二進位可提供FreeBSD的平台。

1.4 版本編號代表什麼意思?

我們並不建議您使用Samba的Alpha版本，除非你曉得並自願從事除錯的工作，許多人取用最新我們建議使用穩定的版本皆能用的稱心如意。假如你是大膽的人，絕對不要冒然不顧一切的協助測試與開發，請別按裝在你們的部門的伺服器上，Samba是非常穩定與安全的，並且應該歸功於許多公開發行的政策。

可何進行編碼:

1.每當軟体有大變動時版本的數字就遞增，例如從1.9.15數字轉移到1.9.16，不過這個版本數字將不會立即出現，並且你們應該繼續使用1.9.15做為生產系統。

2.就在大改變之後被認為尚未穩定，此時會一連串發行Alpha版，例如1.9.16 alpha1，這些者是提供給，熟悉此軟体的專業人士做為測試用途，以alpha的檔案名稱?

3.當Andrew認為alpha版已經穩定了，那麼也就是他開始建議新的使用者按裝的時候，這時他會將以相同版本的數字但去掉alpha字眼，例如1.9.16

4.在穩定的發行版裡發現了無法避免的程式錯誤時就發行輕微修補程式，就以 pxx命名1.9.16p2

發行進度如下：

                1.9.15p7        (生產階段)
                1.9.15p8        (生產階段)
                1.9.16alpha1    (僅供專業人員測試)
                  :
                1.9.16alpha20   (僅供專業人員測試)
                1.9.16          (生產階段)
                1.9.16p1        (生產階段)
